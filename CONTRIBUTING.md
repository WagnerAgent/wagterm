# Contributing

Thanks for your interest in contributing to Wagterm. We welcome issues and PRs from the community.

## Ways to contribute

- Report bugs or propose features via GitHub Issues
- Improve documentation
- Submit bug fixes or new features via Pull Requests

## Before you open an issue

- Search existing issues to avoid duplicates
- Include clear repro steps, expected behavior, and actual behavior
- Share logs or screenshots when possible

## Development setup

```bash
npm install
npm run dev
```

## Pull request guidelines

- Keep PRs focused and small when possible
- Follow the existing code style and lint rules
- Update or add documentation when behavior changes
- If your change affects packaging, confirm `npm run package` still works

## Commit style

- Use clear, descriptive commit messages

## Code of conduct

Be respectful and constructive. Harassment or abuse will not be tolerated.
